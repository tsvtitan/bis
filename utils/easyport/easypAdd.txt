              Как пользоваться этой формой
 1. Выберите нужную операцию в левом окне и нажмите кнопку "Послать" или сделайте двойной клик над выбранной строкой
 2. Вы можете пополнять список дополнительных операций - нажмите "Добавить", напишите или измените значения в верхних окнах и нажмите кнопку "Сохранить". Добавление выполняется после выбранной строки или в конец
 3. Для изменения нажмите кнопку "Изменить", измените значения в верхних окнах (в наименовании операции НЕЛЬЗЯ использовать знак "=") и нажмите кнопку "Сохранить"
 4. Кнопка "Сбросить" восстанавливает значения в верхних окнах
 5. Можно отлаживать команды, изменяя значение во втором окне и затем нажимая на кнопку "Послать" - без запоминания (без нажатия кнопки "Сохранить")
 6. Если в тексте команды указать символ №, то программа запросит ввод числового значения (например, номер) и вставит его вместо символа №
 7. Если в тексте команды указать символ [, то программа запросит ввод строкового значения и вставит его вместо символа [
 8. Если используемые команды устройства уже содержат символы № и  [, то вы можете их заменить на любые другие в INI-файле (подыщите уникальные символы !)
 9. Можно изменить порядок операций в списке - выбрать строку, нажать на одну из стрелок под окном со списком ИЛИ нажать правую кнопку мыши и далее выбрать нужную операцию. Результаты перемещений будут сохранены только при нажатии кнопки или при выполнении пункта "Сохранить"
 10. Запускать команды на исполнение удобно, НЕ закрывая каждый раз окно со списком дополнительных команд - для этого надо установить флажок "Не закрывать форму". При посылке операции в порт форма переместится в правый верхний угол и не будет мешать обзору результатов.
 11. Вы можете сами выбрать положение формы, установив флажок "Не сдвигать форму вправо-вверх". Расположите форму как Вам удобно и посылайте в порт операции.
 12. Чтобы вернуть форму на место, нажмите на нужную "птичку"
 13. Нажав кнопку "Импорт" и указав ini-файл из новой версии.
 можно добавить новые (которых нет) операции (без замены существующих)
 14. Нажав правую кнопку мыши, можно сохранить библиотеку операций
 и/или загрузить её из файла
 15. Вы можете подготовить составную операцию, расположив через
 знак "|" отдельные команды (желательно отладить их заранее !)
 16. Если используемые команды устройства содержат символ |, то вы можете его заменить на любой другой в INI-файле (подыщите уникальный символ !)
 17. Если в тексте команды указать символ ], то программа запросит ввод строкового значения, закодирует введённую строку в  текущей кодировке и вставит его вместо символа ]. Таким образом можно вводить ПО-РУССКИ имя абонента в телефонной книге и/или текст СМС в текстовом режиме (at+cmgf=1)
 18. Если используемые команды устройства содержат символ ], то вы можете его заменить на любой другой в INI-файле (подыщите уникальный символ !)
 19. Вы можете подготовить составную операцию, включив в её состав "внутренние" команды, которые должны начинаться со знака ! (список команд приведён в памятке и со временем будет расширяться)
 20. Если используемые команды устройства содержат символ !, то вы можете его заменить на любой другой в INI-файле (подыщите уникальный символ !)
 21. При отправке СМС можно использовать символ }, означающий, прожатие Enter и символ ~, означающий ввод CtrlZ
 22. Если используемые команды устройства содержат символы } и ~, то вы можете заменить их на любые другие в INI-файле (подыщите уникальные символы !)
 23. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !gets"строка", которая предназначена для ожидания указанной в двойных скобках строки и извлечения поступивших
 после неё из модема-телефона значений (например, номера СМС в памяти)
 - для ссылки на извлечённые значения "аргументы" следует использовать
 - символы %% с идущим следом числом от 1 до 9, которое определит номер
 - части, отделённой запятой или тире(минусом) в полученном из модема ответе
 - см. примеры в операциях
 24. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !waits"строка", которая предназначена для ожидания (!wtime"секунд") указанной в двойных скобках строки и извлечения поступивших после неё данных (см. !gets)
 25. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !wtime"секунд", которая предназначена для установки времени ожидания строки по команде !waits"строка" (по умолчанию 15)
 26. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !pause"секунд", которая предназначена для установки времени задержки выполнения очередной операции (по умолчанию wtime"секунд")
 27. Вы можете Немедленно прекратить выполнение команды или потока команд, ожидания строки по команде !waits"строка", !gets"строка" или паузы по команде !pause"секунд"
 28. Вы можете увеличить (%%<номер от 1 до 9>+<целое>) или уменьшить (%%<номер от 1 до 9>\<целое>), значение парамера, извлекаемого из потока вн. командой !gets - например, %%1\1 или %%2+13
 29. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !pduc"","номер доставки","часов","текст" которая предназначена для для формирования СМС-сообщения в формате PDU=0 (типичен для моделей Siemens)
 30. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" переменную %%pdul, которая предназначена для для формирования длины СМС-сообщения в формате PDU=0 (см. !pduc)
 31. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !cycle"<число повторов>" для зацикливания всей или начальной части  операции (пакета) - допускается ОДНА !cycle на операцию (пакет);  !cycle (с пустым значением числа повторов) задаёт бесконечный цикл);  допускается помещение !cycle ВНУТРЬ пакета (файла);  НЕ допускается включение !bat"пакет" ВНУТРЬ цикла !cycle
 32. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !run"полный путь к модулю>параметр1>параметр2>..." - для запуска  программы с параметрами (параметры разделяются знаком >) - если программа сформирует пакет, можно его выполнить с помощью !bat
 33. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !sound"<полный путь к звуковому файлу>" - для проигрывания  звуковых файлов форматов wav, midi, ...
 34. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !! - для включения любых комментариев
 35. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" переменную %%L - для подстановки имени файла последнего журнала (см. !log)
 36. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" команду !start"чч:мм", которая предназначена для ожидания наступления времени выполнения идущей(их) следом команд(ы) (не более 24 часов)
 37. Вы можете подготовить составную операцию, включив в её состав "внутреннюю" переменную %%i - для подстановки значения номера цикла (при наличии !cycle)
 
       Успехов !
